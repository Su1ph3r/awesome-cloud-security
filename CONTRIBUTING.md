# Contributing to Awesome Cloud Security

Thank you for your interest in contributing to Awesome Cloud Security! This document provides guidelines for contributions.

## How to Contribute

### Adding a New Tool

1. **Fork the repository**
2. **Create a new branch** for your addition
3. **Add the tool** to the appropriate section in README.md
4. **Submit a pull request** with a clear description

### Tool Requirements

Before submitting a tool, please ensure it meets the following criteria:

- **Active Development**: The project should have commits within the last 12 months (unless it's a stable, feature-complete tool)
- **Documentation**: Must have clear documentation or README
- **Open Source**: Preferably open source with a clear license
- **Security Focus**: Must be directly related to cloud security
- **Working State**: The tool should be functional and not abandoned

### Format for Adding Tools

Use the following format when adding tools:

```markdown
- [Tool Name](https://github.com/org/repo) - Brief description of what the tool does.
```

Guidelines for descriptions:
- Keep descriptions concise (one sentence)
- Start with a verb or noun, not "A tool that..."
- Don't include the tool name in the description
- Don't use marketing language

### Categories

Add tools to the most appropriate existing category. If you believe a new category is needed:
1. Open an issue first to discuss
2. New categories should have at least 3 tools
3. Category names should be clear and descriptive

## Pull Request Process

1. **One tool per PR** (preferred) or group related tools
2. **Check for duplicates** before submitting
3. **Verify links** are working
4. **Follow alphabetical order** within sections
5. **Use the PR template** provided

## Code of Conduct

- Be respectful and constructive
- Focus on the technical merit of tools
- Avoid promotional content
- Disclose any conflicts of interest

## Reporting Issues

### Broken Links

If you find a broken link:
1. Open an issue using the "Report Broken Link" template
2. Include the tool name and current URL
3. Suggest an updated URL if available

### Tool Removal Requests

Tools may be removed if they:
- Are no longer maintained (2+ years inactive with no response)
- Contain security vulnerabilities
- No longer function as described
- Violate ethical guidelines

## Questions?

Open an issue for any questions about contributing.

Thank you for helping make this list better!
